the difference between var, let, and const
var: Works inside functions, can be used before declaration, and can be changed
let: Works inside blocks ({}), cannot be used before declaration, but can be changed.
const: Works inside blocks, must be assigned when declared, and cannot be changed
the difference between map(), forEach(), and filter()
map(): Creates a new array by transforming each element.
forEach(): Loops through elements but does not return a new array.
filter(): Creates a new array with only elements that pass a condition.
explain arrow functions and how they are different from regular functions
Arrow functions (=>) are a shorter way to write functions in JavaScript. They don’t have their own this, so they take this from the surrounding scope. Unlike regular functions, arrow functions cannot be used as constructors and don’t have the arguments object. They are great for simple functions but not for methods inside objects.
how JavaScript Promises work
A JavaScript Promise is an object that represents the result of an asynchronous operation (success or failure). It has three states: pending, resolved (fulfilled), or rejected, and uses .then() for success and .catch() for errors.
how closures work in JavaScript
A closure happens when a function remembers and accesses variables from its outer scope even after the outer function has finished running. This allows private variables and maintains state between function calls.
